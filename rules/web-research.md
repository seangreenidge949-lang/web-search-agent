# Web Research — 网络调研调度规则

当用户需要网络调研（搜索多个平台、收集用户评价、竞品对比、趋势监控等）时，按以下流程操作。

## 工作流

```
用户需求（模糊）
    ↓ 解析需求
构建搜索参数
    ↓ 执行搜索
按 rules/web-collector.md 中的流程搜索
    ↓ 回传结果
结果汇总 → 呈现给用户
```

## Step 1: 解析需求

从用户输入中提取：

| 参数 | 来源 | 示例 |
|------|------|------|
| `topic` | 用户说的主题 | "国内 AI 编程工具" |
| `research_goal` | 用户想知道什么 | "用户口碑和工具对比" |
| `keywords_hint` | 从主题推导的关键词 | ["Claude Code", "Cursor", "通义灵码"] |
| `depth` | 根据问题复杂度判断 | quick/standard/deep |
| `language` | 根据主题语境 | "中文为主" |

**depth 判断规则**：
- 简单事实查询 → `quick`（"XX 产品怎么样"）
- 多维度对比 → `standard`（"对比几个工具的优缺点"）
- 全面调研/报告级 → `deep`（"写一份行业分析报告"）

## Step 2: 执行搜索

按照 `rules/web-collector.md` 中定义的搜索流程执行：
1. 分解子主题
2. 选择平台组合
3. 逐条执行 search.py
4. 缺口检查和补搜
5. 整理结果

**search.py 用法**：

```bash
# 搜索特定平台
python3 ./search.py --platform <id> --query "关键词" --limit 10

# 搜索任意网站
python3 ./search.py --site <domain> --query "关键词" --limit 10

# 获取页面评论
python3 ./search.py --comments "页面URL" --limit 20

# 定向抓取全文
python3 ./search.py --scrape "URL" --format text
```

## Step 3: 汇总结果

收到搜索结果后：

1. **统计概览**：X 个平台、Y 条结果、覆盖了哪些来源
2. **关键发现**：按主题维度归类，提炼高互动/高权威的内容
3. **信息缺口**：如果有搜索不到的领域，告知用户并建议补救方式
4. **呈现给用户**：简洁摘要 + 关键原始材料引用

## Step 4: 可选深度分析

如果用户需要更深入的分析（交叉验证、情感分析、结论提炼），可进一步处理搜索结果。

## 关键规则

- **所有搜索都通过 search.py 完成**——不要手动构造 URL 或 API 调用
- **不要猜测搜索结果**——执行搜索后再呈现
- **如果结果不够**——换关键词或平台重搜一轮
